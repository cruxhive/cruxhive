HOW IT WORKS

Four steps. Zero cloud.

CruxHive runs entirely on your machine. Git is the database. SQLite is the index. No accounts, no sync servers, no API costs.

1

Init

One command bootstraps your knowledge layer. CruxHive detects your AI tools and wires itself in automatically.

$ npx cruxhive init
✓ Detected: Claude Code, Cursor
✓ Created .llm/ structure
✓ Registered MCP server
✓ Pulled 12 org entries
2

Propose

AI sessions surface knowledge worth capturing. One command (or one MCP tool call) queues it for review. Secrets are scanned before queuing.

$ cruxhive propose "constraint: never store JWT in localStorage"
✓ Secrets scan: clean
✓ Queued for review
3

Review

Humans always approve before knowledge enters the shared base. AI can propose — never write directly. One keystroke per entry.

$ cruxhive review
» [constraint] never store JWT in localStorage
2 related entries found ↗
[a] approve [r] reject [e] edit
4

Compound

Every approved entry raises the floor for all sessions, all developers, all tools. Month 6 is smarter than month 1 — not because the model improved, but because the knowledge grew.

INFORMATION MODEL

Seven types. One system.

Every piece of knowledge has a declared type. Type determines expiry, confidence defaults, and approval routing.

📌

Fact

Verified truths — IPs, versions, URLs, API contracts. Expires in 90 days.

⚖️

Decision

Architectural choices with rationale. Immutable unless superseded. 365-day review.

🗺️

Plan

Sprint goals, upcoming work. Expires on completion.

🔁

Pattern

Reusable solutions, code conventions the team repeats. 180-day review.

🚫

Constraint

Things the team must NOT do and why. Never expires. NLI-monitored for violations.

🔬

Research

Competitive intelligence, technology evaluations. 90-day review.

✅

Outcome

What actually happened — deploy results, experiment outcomes, post-mortems. Permanent record. Never expires.

FEATURES

Built for the whole team.

Human approval gate

AI proposes, humans decide. Knowledge never enters the shared base without review. The quality gate that prevents AI hallucinations from accumulating.

Hybrid semantic search

BM25 keyword + vector search fused via Reciprocal Rank Fusion. Finds relevant entries even when queries don't share exact words. ~60ms, entirely local.

Tool-agnostic via MCP

One MCP server. Claude Code, OpenCode, Cursor, Copilot, Windsurf, Gemini CLI — all read from the same knowledge base. One update propagates everywhere.

Three-tier model

Personal (~/.cruxhive/) → Project (.llm/ in repo) → Org (shared git remote). Each tier has independent access control. Personal context never leaks to org layer.

Faithfulness detection

Local NLI model detects when AI output contradicts a constraint — post-session, async, zero API cost. Know when your AI ignored what CruxHive told it.

Health dashboard

Coverage, freshness, bus factor, pending queue, faithfulness — visible at a glance. cruxhive ui opens a local web dashboard. Optional, CLI always works.
